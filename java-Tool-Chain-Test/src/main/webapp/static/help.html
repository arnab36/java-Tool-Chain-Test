 Cisco SOW Advisor, build using Watson Developer Cloud APIs, provides a platform for analyzing SOW documents. The tool is browser based and has an easy to use interface where the users can upload and analyze their Statement of Work (SOW) documents. The user can create strategies where he/she specify the mandatory classes and the assign weights to the classes. The SOW document is analyzed against the strategy and a score calculated.

1. Login



This is the login page interface where a user will need to input his/her credentials in order to login. In case the user forgets his/her credentials - they can click on "Having trouble logging in?" to receive further assistance on recovering the credentials.

2. Upload & Analyze



Upload and Analyze , here's where the user uploads the contract that he/she wants to analyse. There is strategy that goes behind analysing different types of contracts, using the "choose strategy" option the user can choose to use a suitable strategy for their contract else they can also use the "create new strategy" option to create a tailor-made strategy for their contract.

3. Create Strategy



Strategy defines list of classes and class properties against which a SOW document should be analyzed. In order to create a new strategy to analyze your SOW you come to this page and follow these steps: 1) Start by giving a Strategy name 2) Then go to the Define Class Tab and select the required class category. 3) Continue with assigning weight, min count, and whether the class is mandatory or not 4) Click on the "done" tab.

4. View Strategy



The view-all strategy page shows the complete list of Strategies with their date of creation, either created by you or other users. Note – You can only view the public access strategies by other users whereas all strategies created by you either public/private can be seen here. Actions you can perform for each strategy: View – applicable for all strategies Copy – applicable for all strategies Edit – applicable only for your own strategies Delete - applicable only for your own strategies

5. Analysis Results







This page comprises mainly of three sections: 1. Score: In this part, you can see the Score Meter which shows the quality of SOW. More the score towards green zone, higher is the SOW quality. Score is a means of evaluating a SOW document. SOW documents are analyzed against a strategy in which a user specifies the presence/absence of classes and the weights he would assign to a class. A SOW document is evaluated against a strategy and a score arrived at – the score varies between 0 and 100. 2. Class Tree In this section, Class Tree is used for analyzing whether all the mandatory classes are present or not. Watson Developer Cloud APIs classify text parsed by Cisco SOW Advisor (CSA) with a certain confidence level, if above threshold, the class label is green otherwise red. We can also see instances only for green class labels. 3. Word Cloud In this section, you see a word-art depiction of all Class Labels present in the SOW. The font-size of these Class Labels is related to the frequency of occurrence of that class.

Classes

NDACompanyRestrict

This is the restrictions of disclosure placed on the COMPANY by the Supplier so the SOW "ideas" cannot be shared

NDASupplierRestrict

This is the restriction of disclosure placed on the SUPPLIER by the Company so the SOW "ideas" cannot be shared

AdherencetoLaws

Clause that says both companies will follow laws and regulations

IntellectualCapital

Any clauses describing Intellectual Capital Protections or assumptions

Compliance

Onboarding, property agreement that either party must COMPLY to

GoverningContract

Clauses that say which contract can govern over another. This classification also includes Master Services Agreement that reference a larger agreement saying the SOW is under that guidance

ChangeControl

This is a Scope Management Language

TMTerms

Explanation of T&M Payment Terms

FPTerms

Explanation of Fixed Price Payment Terms

PaymentTerms

Late payments, early payments, when payments are due, format of payments

CompanyActivities

These are the actions or activities the Company will "do" under the agreement

CompanyRolesandResponsibilities

The Staffing the Company is to bring to the work

SupplierActivities

These are the actions or activities the supplier will "do" under the agreement

SupplierRolesandResponsibilities

The Staffing the Supplier is to Bring to the work

SpecialTerms

Any terms specific to this engagement especially any specific clauses made by Supplier or Cisco as well as any termination terms

TravelTerms

Travel terms
